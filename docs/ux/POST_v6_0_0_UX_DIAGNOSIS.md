# POST-v6.0.0 UX Diagnosis

## Problem

The system had a real backend, live data, Vercel deployment, Supabase runtime, and operator pages, but it still did not feel like a usable visual factory.

## Why

The user had no obvious place to:

- describe a visual task
- speak a request
- create a production request
- see what is currently happening
- see where output appears
- approve or export a visual product

## Conclusion

The system needed a human control cockpit.

## Product rule

A factory UI must answer four human questions:

1. Where do I give the work order?
2. What is the machine doing now?
3. Where are the visual products?
4. What can I approve, regenerate, or export?
